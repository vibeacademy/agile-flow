## Insights and Learnings

### Technical

- **Deploy-first changes are low-risk**: Updating static JSX and markdown docs required zero test changes — the health test checks `{"status":"ok"}` which is stack-independent by design.
- **Search before shipping branding changes**: A quick `grep "Agile Flow Starter"` after the edit would have caught `layout.tsx`. Always search for the old string across the full codebase.

### Process

- **Plan review caught a gap**: The PR review step found the `layout.tsx` miss that the implementation plan overlooked. The two-step workflow (implement then review) provides a safety net even for simple changes.
- **Dry run prep is valuable**: Walking through the updated timeline and docs surfaced concrete concerns (15-min intro window, untested FastAPI swap path) before participants hit them.
